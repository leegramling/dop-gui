# $spec-plan

Use this prompt to refine the roadmap without jumping into large code changes.

## Goal

Convert the current milestone into thin, verifiable steps.

## Instructions

1. Read `plan.md`, `task.md`, and `notes.md`.
2. Identify the active milestone.
3. Split the milestone into small implementation slices.
4. Remove vague wording and replace it with observable outcomes.
5. Update `plan.md`.
6. Set the single next slice in `task.md`.

## Output

- revised milestone steps
- the next thin slice
- any blocked items or dependency risks
